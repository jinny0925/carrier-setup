# Carrier Setup V5

## 주요 기능
- 프로젝트 유형 선택
- 프로젝트별 확인 범위 선택
- 선택 범위의 기본 체크리스트 자동 생성
- 생성 전에 항목 삭제/수정/추가
- 생성 후 항목 추가/삭제/순서 변경
- 기본 항목 다시 불러오기
- 각 체크항목별 상세 관리
  - 상태
  - 확인 내용
  - 배송사/개발팀 답변
  - 운영 반영 내용
  - 근거 자료 링크
  - 비고
  - 확인일
  - 실제 수정 후 수정자/수정일
- 프로젝트별 진행률 및 지연 표시
- 기존 Firestore allowedUsers 로그인 방식 유지

## 적용 방법
1. GitHub의 기존 index.html을 이 파일로 교체합니다.
2. Firestore 규칙을 firestore.rules 내용으로 교체하고 게시합니다.
3. GitHub Pages 배포 후 Ctrl + F5로 새로고침합니다.

## 참고
- 기존 V4 프로젝트는 목록에 표시됩니다.
- 기존 항목에 scopeKey가 없는 경우 기본정보 영역으로 표시될 수 있습니다.
- 현재 템플릿 관리 화면의 수정은 브라우저 메모리에서만 반영됩니다.
  공용 템플릿 저장 기능은 다음 버전에서 Firestore 템플릿 컬렉션으로 연결할 수 있습니다.
- 실제 파일 업로드 대신 OneDrive 등 근거 자료 링크를 저장합니다.
